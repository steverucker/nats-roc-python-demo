Detailed Description

Initializing the SDK.

A ROC application begins with a call to roc_initialize, and ends with a call to roc_finalize. For license management see roc_get_host_id and roc_free_host_id.

Typedef Documentation

typedef const char* roc_host_id

The host ID uniquely identifies a specific machine and is used for license enforcement.

A null-terminated JSON string. Retrieve the host ID for the current machine with roc_get_host_id, free a host ID after use with roc_free_host_id.

Function Documentation

roc_error roc_initialize ( const char *  sdk_path)

Call once at the start of the application before making any other calls to the API.

The sdk_path is the path to the root of the ROC SDK. The root SDK directory should contain the ROC.lic license file, see roc_host_id for details. It is permissible for the SDK directory to be read-only.

Automatic Localization

If sdk_path is NULL, ROC will attempt to locate the SDK automatically relative to:

  1. The application's working directory.
  2. Entries in the PATH environment variable.

Automatic SDK localization will succeed if any of these paths is the SDK path, or a subdirectory thereof.

Example
const char *sdk_path = ...;
roc_initialize(sdk_path);
Parameters
[in]sdk_pathPath to the root of the ROC SDK, or NULL for Automatic Localization.
Remarks
This function is thread-unsafe and must only be called once.

roc_error roc_finalize ( )

Call once at the end of the application after making all other calls to the API.

This will free memory allocated during initialization. Calling roc_initialize after this function to re-initialize the SDK is not supported.

Remarks
This function is thread-unsafe and must only be called once.

roc_error roc_get_host_id ( roc_host_id *  host_id)

Retrieve the roc_host_id of the current machine.

This function provides output identical to running the zlmhostid application included in the bin folder of the SDK. The output of this function (or the zlmhostid application) should be emailed to suppo.nosp@m.rt@r.nosp@m.ankon.nosp@m.e.io to obtain a valid license file prior to calling roc_initialize. Rank One uses the roc_host_id to generate the ROC.lic license file that should be copied to the ROC SDK root directory, overwriting the placeholder license file of the same name.

Free the returned roc_host_id after use with roc_free_host_id.

This function can be called before roc_initialize.

Example
roc_host_id host_id;
roc_get_host_id(&host_id);
Parameters
[out]host_idThe roc_host_id for the current machine.
Remarks
This function is thread-unsafe.
