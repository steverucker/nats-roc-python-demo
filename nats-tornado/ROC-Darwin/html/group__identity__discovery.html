Detailed Description

Group a gallery into identities.

Identities are managed using the roc_template::person_id field. Cluster faces into identities with roc_cluster. Track faces in a video with roc_track. Downsample identities with roc_consolidate. Retrieve identities from a gallery with roc_person_ids.

Construct clusters from a k-nearest neighbors graph.

Clustering groups templates by similarity. Clustering results are stored persistently in the roc_template::person_id field for each template in gallery, and can be retrieved with roc_person_ids. Clusters are sorted by a combination of quality and size, with the largest and most internally similar cluster assigned a roc_template::person_id of zero.

The input to this function is the output from roc_knn.

This function is exposed in the Command Line Interface as roc-cluster.

Aggressiveness

The clustering aggressiveness is a value in the range [-1.0, 1.0] that influences the number of clusters formed. A value toward -1.0 favors more clusters (fewer templates per cluster). Conversely, a value toward 1.0 favors fewer clusters (more templates per cluster). The suggested default value for aggressiveness is 0.0.

Example
roc_gallery gallery = ...;
size_t k = ...;
roc_candidate *neighbors = ...;
roc_cluster(gallery, k, neighbors, 0.0);
Parameters
[in,out]galleryThe gallery used to construct neighbors.
[in]kNumber of nearest neighbors kept for each template in neighbors.
[in]neighborsNearest neighbor graph computed from roc_knn with gallery.
[in]aggressivenessClustering aggressiveness, see Aggressiveness.
Remarks
This function is reentrant.

Track faces across frames in a video.

This is an alternative clustering algorithm that groups templates based on temporal, spatial and facial similarity. Two templates are assigned the same roc_template::person_id if all of the following are true:

  • Their roc_template::timestamp absolute difference is less than or equal to max_time_separation (and greater than zero). The suggested default value for max_time_separation is 1000, corresponding to a one second separation for videos with thirty frames per second.
  • Their detection intersection area divided by union area, or overlap, is greater than or equal to min_detection_overlap. The suggested default value for min_detection_overlap is 0.5.
  • Their facial similarity is greater than or equal to min_similarity. The suggested default value for min_similarity is 0.5.

All templates in the gallery are assumed to belong to the same video.

Tracking results are stored persistently in the roc_template::person_id field for each template in gallery, and can be retrieved with roc_person_ids.

This function is exposed in the Command Line Interface as roc-track.

Maximum Time Separation

The maximum absolute time difference, greater than zero.

Minimum Detection Overlap

The minimum face detection bounding box overlap, between zero and one inclusive.

Minimum Similarity

Minimum template similarity, between zero and one inclusive.

Example

roc_track(gallery, 1000, 0.5, 0.5);

Downsample a gallery by selecting representative templates for each roc_template::person_id.

For each set of templates with the same roc_template::person_id in gallery, this function greedily removes the "most redundant template" until both of the following conditions are met:

  • The maximum pairwise similarity between all remaining templates is less than or equal to max_similarity. See Maximum Similarity.
  • The number of remaining templates is less than or equal to max_count. See Maximum Count.

Templates with roc_template::person_id == ROC_UNKNOWN_PERSON in are retained automatically. This function is generally called after roc_cluster or roc_track to discard superfluous templates.

Consolidation results are stored persistently in the roc_template::person_id field for each template in gallery, and can be retrieved with roc_person_ids.

This function is exposed in the Command Line Interface as roc-consolidate.

Note
This function removes templates from gallery.

Maximum Similarity

The maximum pairwise similarity between templates in a consolidated identity, between zero and one inclusive. After consolidation no two templates belonging to the same person will have a similarity score greater than max_similarity. The suggested default value for max_similarity is 0.9.

Maximum Count

The maximum number of templates in a consolidated identity, greater than zero. After consolidation no person will have more than max_count templates. The suggested default value for max_count is 5.

Example
roc_gallery gallery = ...;
roc_consolidate_gallery(gallery, 0.9, 5);
Parameters
[in,out]galleryThe gallery containing the identities to consolidate.
[in]max_similarityThe maximum pairwise similarity per person, see Maximum Similarity.
[in]max_countThe maximum number of templates per person, see Maximum Count.
Remarks
This function is reentrant.

Retrieve the roc_template::person_id for each template in a gallery.

Used to obtain the results of a call to roc_cluster, roc_track, roc_consolidate.

The roc_person_id at index i in person_ids corresponds to the ith template in gallery, retrievable by roc_at.

Example
roc_gallery gallery = ...;
size_t size;
roc_size(gallery, &size)
roc_person_id *person_ids = (roc_person_id*) malloc(size *
sizeof(roc_person_id));
roc_person_ids(gallery, person_ids);
Parameters
[in]galleryThe gallery to retrieve the roc_template::person_id list from.
[out]person_idsThe roc_template::person_id for each template in gallery.
Remarks
This function is thread-safe.
